Common Use Cases

Base64 encoding is used throughout web development and APIs. Here are the most common scenarios.

🖼️
Embed images in HTML/CSS
Convert images to data URIs (data:image/png;base64,…) to inline them in HTML or CSS without extra HTTP requests.
🔑
HTTP Basic Authentication
Basic Auth headers encode username:password as Base64 in the Authorization header.
📨
Email attachments (MIME)
SMTP email attachments are Base64 encoded using 76-character line-wrapped MIME format as defined in RFC 2045.
🔐
JWT tokens
JSON Web Tokens use URL-safe Base64 (- and _ instead of +//) for the header, payload and signature segments.

KaFinder's Base64 tool handles both encoding and decoding in your browser — no server involved. Upload any file (image, PDF, binary) and get its Base64 string instantly. What is Base64 encoding?
Base64 is a binary-to-text encoding scheme using 64 ASCII characters (A–Z, a–z, 0–9, +, /) plus = for padding. It converts arbitrary binary data into a string safe for transmission over systems that only handle ASCII text, such as email and JSON.
Why does Base64 output end with = or ==?
Base64 encodes 3 bytes at a time into 4 characters. When the input length is not divisible by 3, padding characters (=) are appended to make the output length a multiple of 4. One = means 2 remaining bytes; == means 1 remaining byte.
What is URL-safe Base64?
Standard Base64 uses + and / which have special meaning in URLs. URL-safe Base64 (RFC 4648 §5) replaces these with - and _ so the output can be used in URLs, filenames, and JWT tokens without percent-encoding.
What is MIME Base64?
MIME Base64 (RFC 2045) is identical to standard Base64 but wraps output at 76 characters per line with CRLF line endings. This format is required by email protocols (SMTP) for attachments and message bodies.
Can I encode binary files, not just text?
Yes. Click Upload file to load any file type — images, PDFs, ZIPs. The tool reads it as binary using the FileReader API and encodes it to Base64 entirely in your browser. For images, it also generates a ready-to-embed data URI.
Is my data safe?
Completely. All encoding and decoding runs in JavaScript in your browser. No data is ever sent to KaFinder's servers. Files you upload are read locally via the browser's FileReader API.
